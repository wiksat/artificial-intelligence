W przypadku prostej regresji liniowej (1 zmienna) mamy model postaci $\hat{y} = \alpha x + \beta$, z dwoma parametrami, ktrych bdziemy si uczy. Miar niedopasowania modelu o danych parametrach jest funkcja kosztu (cost function), nazywana te funkcj celu. Najczciej uywa si bdu redniokwadratowego (mean squared error, MSE): $$\large MSE = \frac{1}{N} \sum_{i}^{N} (y - \hat{y})^2 $$

Od jakich $\alpha$ i $\beta$ zacz? W najprostszym wypadku wystarczy po prostu je wylosowa jako niewielkie liczby zmiennoprzecinkowe.

Losowe parametry radz sobie nie najlepiej. Jak lepiej dopasowa nasz prost do danych? Zawsze moemy stara si wyprowadzi rozwizanie analitycznie, i w tym wypadku nawet nam si uda. Jest to jednak szczeglny i do rzadki przypadek, a w szczeglnoci nie bdzie to moliwe w wikszych sieciach neuronowych.

Potrzebna nam bdzie metoda optymalizacji (optimization method), dajc wartoci parametrw minimalizujce dowoln rniczkowaln funkcj kosztu. Zdecydowanie najpopularniejszy jest tutaj spadek wzdu gradientu (gradient descent).

Metoda ta wywodzi si z prostych obserwacji, ktre tutaj przedstawimy. Bardziej szczegowe rozwinicie dla zainteresowanych: sekcja 4.3 "Deep Learning Book", ten praktyczny kurs, analiza oryginalnej publikacji Cauchy'ego (orygina w jzyku francuskim).

Pochodna jest dokadnie rwna granicy funkcji. Dla maego $\epsilon$ mona j przybliy jako: $$\large \frac{f(x)}{dx} \approx \frac{f(x+\epsilon) - f(x)}{\epsilon} $$

Przygldajc si temu rwnaniu widzimy, e:

  • dla funkcji rosncej ($f(x+\epsilon) > f(x)$) wyraenie $\frac{f(x)}{dx}$ bdzie miao znak dodatni
  • dla funkcji malejcej ($f(x+\epsilon) < f(x)$) wyraenie $\frac{f(x)}{dx}$ bdzie miao znak ujemny

Widzimy wic, e potrafimy wskaza kierunek zmniejszenia wartoci funkcji, patrzc na znak pochodnej. Zaobserwowano take, e amplituda wartoci w $\frac{f(x)}{dx}$ jest tym wiksza, im dalej jestemy od minimum (maximum). Pochodna wyznacza wic, w jakim kierunku funkcja najszybciej ronie, za przeciwny zwrot to ten, w ktrym funkcja najszybciej spada.

Stosujc powysze do optymalizacji, mamy: $$\large x_{t+1} = x_{t} - \alpha * \frac{f(x)}{dx} $$

$\alpha$ to niewielka warto (rzdu zwykle $10^{-5}$ - $10^{-2}$), wprowadzona, aby trzyma si zaoenia o maej zmianie parametrw ($\epsilon$). Nazywa si j sta uczc (learning rate) i jest zwykle najwaniejszym hiperparametrem podczas nauki sieci.

Metoda ta zakada, e uywamy caego zbioru danych do aktualizacji parametrw w kadym kroku, co nazywa si po prostu GD (od gradient descent) albo full batch GD. Wtedy kady krok optymalizacji nazywa si epok (epoch).

Im wiksza staa uczca, tym wiksze nasze kroki podczas minimalizacji. Moemy wic uczy szybciej, ale istnieje ryzyko, e bdziemy "przeskakiwa" minima. Mniejsza staa uczca to wolniejszy, ale dokadniejszy trening. Jednak nie zawsze ona pozwala osign lepsze wyniki, bo moe okaza si, e utkniemy w minimum lokalnym. Mona take zmienia sta uczc podczas treningu, co nazywa si learning rate scheduling (LR scheduling). Obrazowo:

Policzmy wic pochodn dla naszej funkcji kosztu MSE. Pochodn liczymy po parametrach naszego modelu, bo to wanie ich chcemy dopasowa tak, eby koszt by jak najmniejszy:

$$\large MSE = \frac{1}{N} \sum_{i}^{N} (y_i - \hat{y_i})^2 $$

W powyszym wzorze tylko $y_i$ jest zaleny od $a$ oraz $b$. Moemy wykorzysta tu regu acuchow (chain rule) i policzy pochodne po naszych parametrach w sposb nastpujcy:

$$\large \frac{\text{d} MSE}{\text{d} a} = \frac{1}{N} \sum_{i}^{N} \frac{\text{d} (y_i - \hat{y_i})^2}{\text{d} \hat{y_i}} \frac{\text{d} \hat{y_i}}{\text{d} a} $$

$$\large \frac{\text{d} MSE}{\text{d} b} = \frac{1}{N} \sum_{i}^{N} \frac{\text{d} (y_i - \hat{y_i})^2}{\text{d} \hat{y_i}} \frac{\text{d} \hat{y_i}}{\text{d} b} $$

Policzmy te pochodne po kolei:

$$\large \frac{\text{d} (y_i - \hat{y_i})^2}{\text{d} \hat{y_i}} = -2 \cdot (y_i - \hat{y_i}) $$

$$\large \frac{\text{d} \hat{y_i}}{\text{d} a} = x_i $$

$$\large \frac{\text{d} \hat{y_i}}{\text{d} b} = 1 $$

czc powysze wyniki dostaniemy:

$$\large \frac{\text{d} MSE}{\text{d} a} = \frac{-2}{N} \sum_{i}^{N} (y_i - \hat{y_i}) \cdot {x_i} $$

$$\large \frac{\text{d} MSE}{\text{d} b} = \frac{-2}{N} \sum_{i}^{N} (y_i - \hat{y_i}) $$

Aktualizacja parametrw wyglda tak:

$$\large a' = a - \alpha * \left( \frac{-2}{N} \sum_{i=1}^N (y_i - \hat{y}_i) \cdot x_i \right) $$ $$\large b' = b - \alpha * \left( \frac{-2}{N} \sum_{i=1}^N (y_i - \hat{y}_i) \right) $$

Liczymy wic pochodn funkcji kosztu, a potem za pomoc reguy acuchowej "cofamy si", dochodzc do tego, jak kady z parametrw wpywa na bd i w jaki sposb powinnimy go zmieni. Nazywa si to propagacj wsteczn (backpropagation) i jest podstawowym mechanizmem umoliwiajcym nauk sieci neuronowych za pomoc spadku wzdu gradientu. Wicej moesz o tym przeczyta tutaj.

def optimize(
    x: np.ndarray, y: np.ndarray, a: float, b: float, learning_rate: float = 0.1
):
    y_hat = a * x + b
    errors = y - y_hat
    y_hat = a * x + b
    errors = y - y_hat
    a = a - learning_rate * (-2 / len(x)) * np.sum(errors * x)
    b = b - learning_rate * (-2 / len(x)) * np.sum(errors)
    return a, b

Trening modeli w PyTorchu jest dosy schematyczny i najczciej rozdziela si go na kilka blokw, dajcych razem ptl uczc (training loop), powtarzan w kadej epoce:

  1. Forward pass - obliczenie predykcji sieci
  2. Loss calculation
  3. Backpropagation - obliczenie pochodnych oraz zerowanie gradientw
  4. Optimalization - aktualizacja wag
  5. Other - ewaluacja na zbiorze walidacyjnym, logging etc.

# initialization
learning_rate = 0.1
a = torch.rand(1, requires_grad=True)
b = torch.rand(1, requires_grad=True)
optimizer = torch.optim.SGD([a, b], lr=learning_rate)
best_loss = float("inf")

# training loop in each epoch
for i in range(1000):
    # forward pass
    y_hat = a * x + b

    # loss calculation
    loss = mse(y, y_hat)

    # backpropagation
    loss.backward()

    # optimization
    optimizer.step()
    optimizer.zero_grad()  # zeroes all gradients - very convenient!

    if i % 100 == 0:
        if loss < best_loss:
            best_model = (a.clone(), b.clone())
            best_loss = loss
        print(f"step {i} loss: {loss.item():.4f}")

print("final loss:", loss)

Przejdziemy teraz do budowy sieci neuronowej do klasyfikacji. Typowo implementuje si j po prostu jako sie dla regresji, ale zwracajc tyle wynikw, ile mamy klas, a potem aplikuje si na tym funkcj sigmoidaln (2 klasy) lub softmax (>2 klasy). W przypadku klasyfikacji binarnej zwraca si czasem tylko 1 warto, przepuszczan przez sigmoid - wtedy wyjcie z sieci to prawdopodobiestwo klasy pozytywnej.

Funkcj kosztu zwykle jest entropia krzyowa (cross-entropy), stosowana te w klasycznej regresji logistycznej. Co wane, sieci neuronowe, nawet tak proste, ucz si szybciej i stabilniej, gdy dane na wejciu (a przynajmniej zmienne numeryczne) s ustandaryzowane (standardized). Operacja ta polega na odjciu redniej i podzieleniu przez odchylenie standardowe (tzw. Z-score transformation).

Uwaga - PyTorch wymaga tensora klas bdcego liczbami zmiennoprzecinkowymi!

from sklearn.metrics import PrecisionRecallDisplay


def get_optimal_threshold(
    precisions: np.array, 
    recalls: np.array, 
    thresholds: np.array
) -> Tuple[int, float]:
    
    numerator = 2 * precisions * recalls
    denominator = precisions + recalls
    f1_scores = np.divide(numerator, denominator, out=np.zeros_like(numerator), where=denominator != 0)
    
    optimal_idx = np.argmax(f1_scores)
    optimal_threshold = thresholds[optimal_idx]
    
    return optimal_idx, optimal_threshold


def plot_precision_recall_curve(y_true, y_pred_score) -> None:
    precisions, recalls, thresholds = precision_recall_curve(y_true, y_pred_score)
    optimal_idx, optimal_threshold = get_optimal_threshold(precisions, recalls, thresholds)

    disp = PrecisionRecallDisplay(precisions, recalls)
    disp.plot()
    plt.title(f"Precision-recall curve (opt. thresh.: {optimal_threshold:.4f})")
    plt.axvline(recalls[optimal_idx], color="green", linestyle="-.")
    plt.axhline(precisions[optimal_idx], color="green", linestyle="-.")
    plt.show()

Sieci neuronowe

Wszystko zaczo si od inspirowanych biologi sztucznych neuronw, ktrych prbowano uy do symulacji mzgu. Naukowcy szybko odeszli od tego podejcia (sam problem modelowania okaza si te znacznie trudniejszy, ni sdzono), zamiast tego uywajc neuronw jako jednostek reprezentujc dowoln funkcj parametryczn $f(x, \Theta)$. Kady neuron jest zatem bardzo elastyczny, bo jedyne wymagania to funkcja rniczkowalna, a mamy do tego wektor parametrw $\Theta$.

W praktyce najczciej mona spotka si z kilkoma rodzinami sieci neuronowych:

  1. Perceptrony wielowarstwowe (MultiLayer Perceptron, MLP) - najbardziej podobne do powyszego opisu, niezbdne do klasyfikacji i regresji
  2. Konwolucyjne (Convolutional Neural Networks, CNNs) - do przetwarzania danych z zalenociami przestrzennymi, np. obrazw czy dwiku
  3. Rekurencyjne (Recurrent Neural Networks, RNNs) - do przetwarzania danych z zalenociami sekwencyjnymi, np. szeregi czasowe, oraz kiedy do jzyka naturalnego
  4. Transformacyjne (Transformers), oparte o mechanizm atencji (attention) - do przetwarzania jzyka naturalnego (NLP), z ktrego wypary RNNs, a coraz czciej take do wszelkich innych danych, np. obrazw, dwiku
  5. Grafowe (Graph Neural Networks, GNNS) - do przetwarzania grafw

Na tym laboratorium skupimy si na najprostszej architekturze, czyli MLP. Jest ona powszechnie czona z wszelkimi innymi architekturami, bo pozwala dokonywa klasyfikacji i regresji. Przykadowo, klasyfikacja obrazw to zwykle CNN + MLP, klasyfikacja tekstw to transformer + MLP, a regresja na grafach to GNN + MLP.

Dodatkowo, pomimo prostoty MLP s bardzo potne - udowodniono, e perceptrony (ich powszechna nazwa) s uniwersalnym aproksymatorem, bdcym w stanie przybliy dowoln funkcj z odpowiednio maym bdem, zakadajc wystarczajc wielko warstw sieci. Szczeglne ich wersje potrafi nawet reprezentowa drzewa decyzyjne.

Sieci MLP

Dla przypomnienia, na wejciu mamy punkty ze zbioru treningowego, czyli $d$-wymiarowe wektory. W klasyfikacji chcemy znale granic decyzyjn, czyli krzyw, ktra oddzieli od siebie klasy. W wejciowej przestrzeni moe by to trudne, bo chmury punktw z poszczeglnych klas mog by ze sob do pomieszane. Pamitajmy te, e regresja logistyczna jest klasyfikatorem liniowym, czyli w danej przestrzeni potrafi oddzieli punkty tylko lini prost.

Sie MLP skada si z warstw. Kada z nich dokonuje nieliniowego przeksztacenia przestrzeni (mona o tym myle jak o skadaniu przestrzeni jak prost/aman), tak, aby w finalnej przestrzeni nasze punkty byy moliwie liniowo separowalne. Wtedy ostatnia warstwa z sigmoid bdzie potrafia je rozdzieli od siebie.

Poszczeglne neurony skadaj si z iloczynu skalarnego wej z wagami neuronu, oraz nieliniowej funkcji aktywacji. W PyTorchu s to osobne obiekty - nn.Linear oraz np. nn.Sigmoid. Funkcja aktywacji przyjmuje wynik iloczynu skalarnego i przeksztaca go, aby sprawdzi, jak mocno reaguje neuron na dane wejcie. Musi by nieliniowa z dwch powodw. Po pierwsze, tylko nieliniowe przeksztacenia s na tyle potne, eby umoliwi liniow separacj danych w ostatniej warstwie. Po drugie, liniowe przeksztacenia zwyczajnie nie dziaaj. Aby zrozumie czemu, trzeba zobaczy, co matematycznie oznacza sie MLP.

Zapisane matematycznie MLP to:

$\large h_1 = f_1(x) \\ h_2 = f_2(h_1) \\ h_3 = f_3(h_2) \\ ... \\ h_n = f_n(h_{n-1}) $

gdzie $x$ to wejcie $f_i$ to funkcja aktywacji $i$-tej warstwy, a $h_i$ to wyjcie $i$-tej warstwy, nazywane ukryt reprezentacj (hidden representation), lub latent representation. Nazwa bierze si z tego, e w rodku sieci wycigamy cechy i wzorce w danych, ktre nie s widoczne na pierwszy rzut oka na wejciu.

Zamy, e uczymy si na danych $x$ o jednym wymiarze (dla uproszczenia wzorw) oraz nie mamy funkcji aktywacji, czyli wykorzystujemy tak naprawd aktywacj liniow $f(x) = x$. Zobaczmy jak bd wyglda dane przechodzc przez kolejne warstwy:

$\large h_1 = f_1(xw_1) = xw_1 \\ h_2 = f_2(h_1w_2) = xw_1w_2 \\ ... \\ h_n = f_n(h_{n-1}w_n) = xw_1w_2...w_n $

gdzie $w_i$ to jest parametr $i$-tej warstwy sieci, $x$ to s dane (w naszym przypadku jedna liczba) wejciowa, a $h_i$ to wyjcie $i$-tej warstwy.

Jak wida, taka sie o $n$ warstwach jest rwnowana sieci o jednej warstwie z parametrem $w = w_1w_2...w_n$. Wynika to z tego, e zoenie funkcji liniowych jest take funkcj liniow - patrz notatki z algebry :)

Jeeli natomiast uyjemy nieliniowej funkcji aktywacji, czsto oznaczanej jako $\sigma$, to wszystko bdzie dziaa. Co wane, ostatnia warstwa, dajca wyjcie sieci, ma zwykle inn aktywacj od warstw wewntrz sieci, bo te ma inne zadanie - zwrci warto dla klasyfikacji lub regresji. Na wyjciu korzysta si z funkcji liniowej (regresja), sigmoidalnej (klasyfikacja binarna) lub softmax (klasyfikacja wieloklasowa).

Wewntrz sieci uywano kiedy sigmoidy oraz tangensa hiperbolicznego tanh, ale okazao si to nieefektywne przy uczeniu gbokich sieci o wielu warstwach. Nowoczesne sieci korzystaj zwykle z funkcji ReLU (rectified linear unit), ktra jest zaskakujco prosta: $ReLU(x) = \max(0, x)$. Okazao si, e bardzo dobrze nadaje si do treningu nawet bardzo gbokich sieci neuronowych. Nowsze funkcje aktywacji s gwnie modyfikacjami ReLU.

MLP w PyTorchu

Warstw neuronw w MLP nazywa si warstw gst (dense layer) lub warstw w peni poczon (fully-connected layer), i taki opis oznacza zwykle same neurony oraz funkcj aktywacji. PyTorch, jak ju widzielimy, definiuje osobno transformacj liniow oraz aktywacj, a wic jedna warstwa skada si de facto z 2 obiektw, wywoywanych jeden po drugim. Inne frameworki, szczeglnie wysokopoziomowe (np. Keras) cz to czsto w jeden obiekt.

MLP skada si zatem z sekwencji obiektw, ktre potem wywouje si jeden po drugim, gdzie wyjcie poprzedniego to wejcie kolejnego. Ale nie mona tutaj uywa Pythonowych list! Z perspektywy PyTorcha to wtedy niezalene obiekty i nie zostanie wtedy przekazany midzy nimi gradient. Trzeba tutaj skorzysta z nn.Sequential, aby tworzy taki pipeline.

Rozmiary wejcia i wyjcia dla kadej warstwy trzeba w PyTorchu podawa explicite. Jest to po pierwsze edukacyjne, a po drugie czsto uatwia wnioskowanie o dziaaniu sieci oraz jej debugowanie - mamy jasno podane, czego oczekujemy. Niektre frameworki (np. Keras) obliczaj to automatycznie.

Co wane, ostatnia warstwa zwykle nie ma funkcji aktywacji. Wynika to z tego, e obliczanie wielu funkcji kosztu (np. entropii krzyowej) na aktywacjach jest czsto niestabilne numerycznie. Z tego powodu PyTorch oferuje funkcje kosztu zawierajce w rodku aktywacj dla ostatniej warstwy, a ich implementacje s stabilne numerycznie. Przykadowo, nn.BCELoss przyjmuje wejcie z zaaplikowanymi ju aktywacjami, ale moe skutkowa under/overflow, natomiast nn.BCEWithLogitsLoss przyjmuje wejcie bez aktywacji, a w rodku ma specjaln implementacj czc binarn entropi krzyow z aktywacj sigmoidaln. Oczywicie w zwizku z tym aby dokona potem predykcji w praktyce, trzeba pamita o uyciu funkcji aktywacji. Czsto korzysta si przy tym z funkcji z moduu torch.nn.functional, ktre s w tym wypadku nieco wygodniejsze od klas wywoywalnych z torch.nn.

Cae sieci w PyTorchu tworzy si jako klasy dziedziczce po nn.Module. Co wane, obiekty, z ktrych tworzymy sie, np. nn.Linear, take dziedzicz po tej klasie. Pozwala to na bardzo moduow budow kodu, zgodn z zasadami OOP. W konstruktorze najpierw trzeba zawsze wywoa konstruktor rodzica - super().__init__(), a pniej tworzy si potrzebne obiekty i zapisuje jako atrybuty. Kady atrybut dziedziczcy po nn.Module lub nn.Parameter jest uwaany za taki, ktry zawiera parametry sieci, a wic przy wywoaniu metody parameters() - parametry z tych atrybutw pojawi si w licie wszystkich parametrw. Musimy te zdefiniowa metod forward(), ktra przyjmuje tensor x i zwraca wynik. Typowo ta metoda po prostu uywa obiektw zdefiniowanych w konstruktorze.

Regularyzacja specyficzna dla sieci neuronowych to dropout. Polega on na losowym wyczaniu zadanego procenta neuronw podczas treningu. Pomimo prostoty okazaa si niesamowicie skuteczna, szczeglnie w treningu bardzo gbokich sieci. Podejcie to mona potraktowa jak ensemble learning, podobny do lasw losowych - wyczajc losowe czci sieci, w kadej iteracji trenujemy nieco inn sie, co odpowiada urednianiu predykcji rnych algorytmw. Ostatni, a by moe najwaniejszy rodzaj regularyzacji to wczesny stop (early stopping). W kadym kroku mocniej dostosowujemy terenow sie do zbioru treningowego, a wic zbyt dugi trening bdzie skutkowa przeuczeniem. W metodzie wczesnego stopu uywamy wydzielonego zbioru walidacyjnego (pojedynczego, metoda holdout), sprawdzajc co okrelon liczb epok wynik na tym zbiorze. Okres, przez ktry czekamy na uzyskanie lepszego wyniku, to cierpliwo (patience). Im mniejsze, tym mocniejszy jest ten rodzaj regularyzacji, ale trzeba z tym uwaa, bo atwo jest przesadzi i zbyt szybko przerywa trening. Niektre implementacje uwzgldniaj tzw. grace period, czyli gwarantowan minimaln liczb epok, przez ktr bdziemy trenowa sie, niezalenie od wybranej cierpliwoci.

Dodatkowo ryzyko przeuczenia mona zmniejszy, uywajc mniejszej staej uczcej.

Opisalimy wczeniej podstawowy optymalizator w sieciach neuronowych - spadek wzdu gradientu. Jednak wymaga on uycia caego zbioru danych, aby obliczy gradient, co jest czsto niewykonalne przez rozmiar zbioru. Dlatego wymylono stochastyczny spadek wzdu gradientu (stochastic gradient descent, SGD), w ktrym uywamy 1 przykadu naraz, liczymy gradient tylko po nim i aktualizujemy parametry. Jest to oczywicie do grube przyblienie gradientu, ale pozwala robi szybko duo maych krokw. Kompromisem, ktrego uywa si w praktyce, jest minibatch gradient descent, czyli uywanie batchy np. 32, 64 czy 128 przykadw.

Rzadko wspominanym, a wanym faktem jest take to, e stochastyczno metody optymalizacji jest sama w sobie te metod regularyzacji, a wic batch_size to take hiperparametr.

Obecnie najpopularniejsz odmian SGD jest Adam, gdy uczy on szybko sie oraz daje bardzo dobre wyniki nawet przy niekoniecznie idealnie dobranych hiperparametrach. W PyTorchu najlepiej korzysta z jego implementacji AdamW, ktra jest nieco lepsza ni implementacja Adam. Jest to zasadniczo zawsze wybr domylny przy treningu wspczesnych sieci neuronowych.

Na koniec laboratorium dooymy do naszego modelu jeszcze 3 powrzechnie uywane techniki, ktre s bardzo proste, a pozwalaj czsto ulepszy wynik modelu.

Pierwsz z nich s warstwy normalizacji (normalization layers). Powstay one pocztkowo z zaoeniem, e przez przeksztacenia przestrzeni dokonywane przez sie zmienia si rozkad prawdopodobiestw pomidzy warstwami, czyli tzw. internal covariate shift. Pniej okazao si, e zastosowanie takiej normalizacji wygadza powierzchni funkcji kosztu, co uatwia i przyspiesza optymalizacj. Najpowszechniej uywan normalizacj jest batch normalization (batch norm).

Drugim ulepszeniem jest dodanie wag klas (class weights). Mamy do czynienia z problemem klasyfikacji niezbalansowanej, wic klasa mniejszociowa, waniejsza dla nas, powinna dosta wiksz wag. Implementuje si to trywialnie prosto - po prostu mnoymy warto funkcji kosztu dla danego przykadu przez wag dla prawdziwej klasy tego przykadu. Praktycznie kady klasyfikator operujcy na jakiej waonej funkcji moe dziaa w ten sposb, nie tylko sieci neuronowe.

Ostatnim ulepszeniem jest zamiana SGD na optymalizator Adam, a konkretnie na optymalizator AdamW. Jest to przykad optymalizatora adaptacyjnego (adaptive optimizer), ktry potrafi zaadaptowa sta uczc dla kadego parametru z osobna w trakcie treningu. Wykorzystuje do tego gradienty - w uproszczeniu, im wiksza wariancja gradientu, tym mniejsze kroki w tym kierunku robimy.

class NormalizingMLP(nn.Module):
    def __init__(self, input_size: int, dropout_p: float = 0.5):
        super().__init__()

        self.first_layer = nn.Linear(input_size, 256)
        self.second_layer = nn.Linear(256, 128) 
        self.third_layer = nn.Linear(128, 1)
        self.mlp=nn.Sequential(
            self.first_layer,
            nn.BatchNorm1d(256),
            nn.ReLU(),
            nn.Dropout(dropout_p),
            self.second_layer,
            nn.BatchNorm1d(128),
            nn.ReLU(),
            nn.Dropout(dropout_p),
            self.third_layer
        )
    
    def forward(self, x):
        return self.mlp(x)

    def predict_proba(self, x):
        return sigmoid(self(x))
    
    def predict(self, x, threshold: float = 0.5):
        y_pred_score = self.predict_proba(x)
        return (y_pred_score > threshold).to(torch.int32)

  • staa uczca
  • batch size

  • prawdopodobiestwo dropoutu
